﻿SERVER 0.4.1

Теперь всё, что связано с сервером, вынесено в отдельную папку SERVER.

Что лежит тут:
- BUILD_SERVER_PACKAGE.bat — сборка серверного архива из веб-версии
- READY — готовые серверные архивы и папки
- SERVER_ENV.txt — настройки сервера по умолчанию
- server.js — простой Node-сервер для статической раздачи dist
- AUTO_UNPACK_AND_START.sh — серверный скрипт для авто-распаковки / авто-перезапуска

Как это работает сейчас:
1) Если в KEY_API не вписаны облачные ключи и серверные строки — это НОРМАЛЬНО.
2) Приложение остаётся в локальном режиме.
3) Android APK всё равно ставится локально на телефон через adb.
4) Когда понадобится сервер — собираешь серверный пакет через BUILD_SERVER_PACKAGE.bat.

Что делает BUILD_SERVER_PACKAGE.bat:
- читает KEY_API
- обновляет .env.local
- собирает web dist
- складывает серверный пакет в SERVER\READY
- делает zip, который потом можно загрузить на сервер

Что делать на сервере потом:
1) загружаешь содержимое серверного пакета или zip
2) распаковываешь
3) запускаешь AUTO_UNPACK_AND_START.sh
4) сервер сам создаёт releases/current и поднимает node server.js

Важно:
- Для этого маршрута нужен Node.js на сервере.
- Если есть pm2 — скрипт использует его.
- Если pm2 нет — скрипт стартует сервер через nohup.
